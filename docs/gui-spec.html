GUI 画面定義書

Juml の Swing GUI（VS Code 風タブ中心モデル）の画面仕様。位置づけ・画面項目・振る舞い・メッセージ・非機能要件を、実装（juml.app.uml）に基づいて定義します。

対象: GUI 画面 主 1 + ダイアログ 11 + パネル 6 操作手順は GUI マニュアル
本書の位置づけ: 操作の流れ・ショートカット・図種カタログは GUI 操作マニュアル、内部実装・クラス責務は 開発者リファレンスを参照。本書は「画面定義書（項目・属性・バリデーション・メッセージ）」に特化します。文言・初期値・範囲は src/main/java/juml/app/uml/ 配下の実装に準拠しています。

画面の目的・概要

Juml の GUI は、Java / Android（Gradle + AndroidManifest.xml + AIDL）プロジェクトを解析し、PlantUML 経由で 15 種類の UML 図を生成・閲覧・エクスポートするデスクトップアプリ（Java Swing 製）です。

  • 主目的: ソースコードを読み込み、クラス図・シーケンス図・パッケージ図などを対話的に閲覧し、設計理解・影響調査・ドキュメント化を支援する。
  • UI モデル: 「タブ＝エディタ」の VS Code 風モデル。1 つの図（題材 + 図種）が 1 タブに対応し、左ペインのツリー（ナビゲータ）から開く。特別な「Home 画面」は持たない。
  • 主画面構成: 単一のメインウィンドウ（UmlMainFrame）＋ モーダルダイアログ群（プロジェクト選択・スコープ・スタイル・検索など）＋ 右ペイン固定のユーティリティパネル群（Impact / References / Func Diff / Functions / Members / Manifest）。

利用者・権限・ロール（誰が使えるか）

Juml は単一ユーザのローカルデスクトップアプリで、ログイン・認証・サーバ側権限といった概念は持ちません。すべての機能はアプリを起動できる利用者（開発者）が等しく利用できます。

観点仕様
ユーザ区分 / ロール区分なし（全機能が単一ロールで利用可能）。アプリ内に権限制御は存在しない。
認証なし（ログイン画面なし）。
実行に必要な権限OS 上で対象プロジェクトフォルダの読み取り権限、エクスポート先の書き込み権限。Java 17+ 実行環境。
データの保存先プロジェクト設定・ウィンドウ位置・解析キャッシュはローカル（SQLite / ディスクキャッシュ）。外部送信なし。
破壊的操作解析は読み取りのみで対象ソースを変更しない。「一覧から削除」も最近リストの除去のみでディスク上のファイルは削除しない。

画面の状態・モード（新規 / 編集 / 参照など）

Juml は文書を「新規作成 / 編集」するアプリではなく、ソースを解析して図を閲覧・出力するアプリです。モードは「アプリ全体の状態」と「各タブの状態」の二層で管理されます。

アプリ全体の状態

状態説明主な操作可否
空（No Project）プロジェクト未ロード図操作は不可（多くは「Open a project first.」通知）。Open / Preferences / Help は可。
解析中（Analyzing）バックグラウンド解析（SwingWorker）。全面ローディングオーバーレイ表示。UI 操作はオーバーレイで抑止。Cancel Loading のみ有効。
図表示（Diagram Shown）解析完了・図タブ表示中全機能利用可。タブ操作・図種切替・ズームは状態を変えない内部遷移。

タブの状態（参照モード中心）

状態説明
未生成（None）当該キーのタブが存在しない
描画中（Rendering）SwingWorker による非同期描画中（"Rendering..." 表示）
表示 / フォーカス（Shown）描画成功。アクティブなタブ。同一キー再要求はフォーカスのみ
非フォーカス（Background）他タブ選択中。状態は保持され再選択で復帰
エラー表示（Error）描画失敗。タブ内に原因と対処を表示
図・スタイル設定の編集は「設定ダイアログ → アクティブ／全タブ再描画」という形で行われ、図そのものを直接編集する WYSIWYG 編集モードは存在しません（出力は PlantUML / SVG / PNG）。

振る舞い — 遷移先・後続処理

状態の流れの詳細（状態機械図・シーケンス図）は GUI マニュアルの状態遷移を参照。要点は次のとおり。

  • プロジェクトロード成功: ツリー構築 → 既定 Package 図を 1 タブ開く → 図表示状態へ。
  • タブのフォーカス: mirrorToState() でグローバル状態を更新し、ツリーのハイライト・ツールバー図種・ステータスへ逆同期（suppressNotify で循環を防止）。
  • スタイル / スコープ変更: アクティブ（または全）タブを再描画し、結果を再びグローバル状態へ反映。
  • ダイアログの結果: 起点・スコープ・layout / navigation キーなどを設定し、対応するタブを開く / 再描画する。
  • タブを閉じる / プロジェクト再オープン: タブ状態は破棄。再オープン時はアクティブタブ状態をクリアし、スタイル等のプロジェクト設定は永続化から復元。

例外・非機能 — 描画失敗・エラー時の挙動

  • 描画失敗: タブ内に「Couldn't render this diagram.」と原因・対処を表示し、下ペインに生成 PlantUML を残す。失敗してもアプリは継続。
  • Graphviz 無効時: 「Graphviz が無効のため、純 Java の Smetana レイアウトを使用しています。Smetana は大きな図を描画できないことがあります。」と表示し、Diagram → Enable Graphviz (dot)… での有効化、または図を小さくする手順（単一パッケージ/クラス/メソッド選択・Preset・Scope）を案内。
  • Graphviz 有効・巨大図: 「The diagram may be too large for the layout engine.」として同様の縮小手順を案内。
  • PNG エクスポートの上限: キャンバス上限（既定 16384）で scale max 自動縮小。収まらない場合は SVG 出力を推奨。
  • 解析エラー: 「Failed to analyze project: …」ダイアログを表示し「空」状態へ戻る。キャンセル時はステータスに「Cancelled.」。
  • 設定保存: プロジェクト設定保存はベストエフォート（例外は握りつぶしてアプリ継続）。

例外・非機能 — パフォーマンス

  • 非同期解析: プロジェクト解析・図描画・影響解析・差分比較はいずれも SwingWorker で背景実行し、EDT（UI スレッド）をブロックしない。
  • 進捗スロットリング: 解析進捗の UI 反映は約 150ms 単位でスロットルし、過度な再描画を防ぐ。
  • 遅延生成・遅延展開: ツリーはパッケージ→クラス→メソッドを展開時に構築。Functions / Members タブはタブ選択時に初めて生成。
  • キャッシュ: 解析結果（メモリ + ディスク）・参照インデックスを再利用。Clear Analysis Cache で明示的に再構築。
  • 軽量モード: Scope の Headers only (light) や Max classes・Preset で大規模プロジェクトの描画コストを抑制。
  • 巨大図対応: SVG モードは PlantUML の 4096×4096 PNG キャンバス制限を回避。Graphviz (dot) でより堅牢なレイアウトに切替可能。

例外・非機能 — レスポンシブ / ウィンドウ

  • 分割ペイン: 左右（ツリー / タブ）と上下（プレビュー / ソース）の JSplitPane をユーザがドラッグでリサイズ可能。初期比は左右 0.22 / 上下 0.85。
  • ウィンドウサイズ永続化: 既定 1200×800。前回サイズ・位置を保存し次回起動時に復元。
  • ズーム / パン: プレビューは 0.1〜8.0 倍。Ctrl+ホイールはポインタ基点ズーム、通常ホイールは親スクロールへ委譲。Zoom to Fit はビューポートに合わせて自動調整。
  • ドキュメントサイト: 本ドキュメント（HTML）はサイドバー折りたたみ・テーブルのカード化など、880px 以下のモバイル幅にも対応。

例外・非機能 — アクセシビリティ

  • キーボード操作: 主要操作にショートカット（Ctrl+O/S・F5・Ctrl+Shift+F・Ctrl+1/2/3・Ctrl+=/-/0/F・Ctrl+,・F1）。メニューにニーモニックを設定。
  • ダイアログ共通: Esc でキャンセル / クローズ、Enter で OK（デフォルトボタン）。検索系はフィルタ欄で ↓ により一覧へフォーカス移動、Enter / ダブルクリックで確定。開いた直後にフィルタ欄へ自動フォーカス。
  • プレビュー: フォーカス可能で Ctrl+A によるテキスト全コピーに対応。リンク領域上ではハンドカーソルに変化。
  • 日本語フォント: フォント未指定時は実行環境の日本語フォントを自動検出し、文字化け（豆腐 □）を防止。Style Settings のプレビューで日本語表示可否を注記。
  • 視認性: 線種 Ortho・影オフ・間隔調整・テーマ切替や Optimize for Readability により可読性を調整可能。
