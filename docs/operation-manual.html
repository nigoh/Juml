操作マニュアル

入手・インストール（exe / jar / ビルド）から GUI・CLI の実運用までを一気通貫で。コマンド例を多めに収録しています。

対象: 少しコーディングできる人 Java 17+ exe は JRE 同梱 図種 15+

対象読者と前提

このマニュアルは ターミナルで java -jar や ./gradlew を叩ける程度のコーディング経験がある方を対象に、Juml を「手元で動かして UML 図を作る」ための操作を順を追って説明します。GUI の細かな画面仕様や CLI 全フラグはそれぞれ GUI マニュアル / 開発者リファレンス に譲り、ここでは 入手 → 起動 → 図を出すまでの導線に集中します。

前提環境: 図を作るだけなら追加インストールは不要です。PlantUML は同梱（Smetana レイアウト）で、Graphviz / PlantUML を別途入れる必要はありません。exe 版なら Java すら不要です。

Juml で何ができるか

Juml は Java / Android（Gradle + AndroidManifest.xml + AIDL）プロジェクトを解析して UML 図を生成する Swing GUI + CLI ツールです。クラス図・シーケンス図・パッケージ図のほか、Android / AOSP 向けの Manifest 図・依存グラフ・Soong 図など 全 15 種以上を出力できます。出力は SVG（推奨）/ PNG / PlantUML テキスト(.puml) / Markdown サマリー。

使い分け: 対話的に探索したいときは GUI、CI やスクリプトで一括出力したいときは CLI。どちらも同じ解析エンジンを使います。

入手とインストール — 3 つの実行形態

用途に合わせて 3 つの形態から選べます。

形態Java の要否向いている人入手方法
Windows exe（自己完結）不要（JRE 同梱）とにかく動かしたい / 配布したいRelease の Juml-windows-*.zip
jar（fat jar）Java 17+ が必要OS 問わず使いたい / CLI で回したいRelease の Juml.jar
ソースからビルドJDK 17+ が必要改造したい / 最新を試したいgit clone → ./gradlew

Windows exe（推奨・Java 不要）

GitHub Release から Juml-windows-<version>.zip をダウンロードして展開します。中の Juml/ フォルダに Juml.exe と専用 runtime\（JRE）が入っており、Java をインストールしていない PC でもそのまま動きます。

# 1) zip を展開（任意のフォルダで可）
# 2) Juml.exe をダブルクリック → プロジェクト選択ダイアログが開く

# コマンドラインから起動（プロジェクトを渡すと起動時に解析）
Juml.exe C:\path\to\AndroidStudioProjects\MyApp
この exe は JDK 標準の jpackage で生成した app-image です。インストーラ（.msi）ではなく 展開して使うポータブル形式なので、フォルダごとコピー・削除できます。

jar 版（Java 17+）

OS を問わず使える単一ファイルの fat jar です。Java 17 以上（JRE / JDK どちらでも可）が必要です。

# バージョン確認（17 以上であること）
java -version

# GUI を起動（引数なし＝プロジェクト選択ダイアログ）
java -jar Juml.jar

# プロジェクトを渡して起動時に解析
java -jar Juml.jar ~/AndroidStudioProjects/MyApp

# ヘルプ（CLI 全オプション）
java -jar Juml.jar --help

ソースからビルド

改造したい・最新の main を試したい場合は JDK 17 以上でビルドします。Gradle Wrapper 同梱なので Gradle のインストールは不要です。

# クローン
git clone https://github.com/nigoh/Juml.git
cd Juml

# fat jar をビルド → build/libs/Juml.jar
./gradlew jar

# 配布 zip（jar + 起動スクリプト + 既定設定）→ build/distributions/juml-*.zip
./gradlew makeZip

# 自己完結ネイティブイメージ（JRE 同梱）を生成
#   Windows で実行  → build/jpackage/Juml/Juml.exe
#   Linux で実行    → build/jpackage/Juml/bin/Juml
#   macOS で実行    → build/jpackage/Juml.app
./gradlew jpackageImage

# 上記を OS 別サフィックス付き zip に固める → build/distributions/Juml--.zip
./gradlew jpackageZip
exe を作るには JDK が必要: jpackageImage は JDK 同梱の jpackage を使います。JRE で実行すると「jpackage が見つかりません」とエラーになるので、JAVA_HOME を JDK 17 以上に向けてください。また jpackage は 実行している OS 向けのイメージしか作れません。配布用の Windows .exe は Windows 上（または CI の Windows ランナー）で生成します。
CI でも自動生成: タグ v* を push すると GitHub Actions が jar・配布 zip に加えて Windows ランナーで Juml-windows-*.zip（exe 同梱）を生成し、同じ Release に自動添付します（.github/workflows/release.yml）。

GUI の基本ワークフロー

起動後の基本的な流れは次の 4 ステップです。画面構成やショートカットの詳細は GUI マニュアル を参照してください。

  1. プロジェクトを開く — File → Open Project（または起動時に引数で渡す）。Android Studio / Gradle プロジェクトのルートを選ぶと解析が走ります。
  2. 左ツリーで対象を選ぶ — クラス / パッケージ / モジュール / メソッドをクリック。VS Code のように 1 つの題材＋図種＝1 タブ（エディタ）として開きます。
  3. 図種を切り替える — ツールバー下段の図種トグル、または Diagram メニューでアクティブタブの図種を変更。
  4. 図を保存／エクスポート — 図を右クリック、または File メニューから SVG / PNG / PUML / Markdown で書き出し。
連動の仕組み: アクティブタブ ⇄ 左ツリーのハイライト ⇄ ツールバー / メニューの図種表示 ⇄ ステータスバーは常に同期します。「いまフォーカスしているタブ」に対して操作が効く、と覚えてください。

図種早見表（GUI ⇄ CLI）

主な図種と、GUI での起点・対応する CLI フラグの対応です。CLI フラグが空欄の図種は主に GUI から利用します。図種カタログの詳細は 開発者リファレンス を参照してください。

図種GUI での起点CLI フラグ用途
クラス図 (Class)クラス / パッケージを選択-c構造・継承・依存の俯瞰
シーケンス図 (Sequence)メソッドを選択-q Class.method呼び出しトレース（if→alt / loop）
アクティビティ図 (Activity)メソッドを選択 → 図種切替—メソッド内の制御フロー
コールグラフ (Call Graph)メソッドを選択 → 図種切替—どの関数を呼ぶか
パッケージ図 (Package)図種切替—パッケージ単位の依存
継承図 (Inheritance)図種切替—extends / implements 階層
共通クラス図 (Common)図種切替—参照されやすいハブクラス上位
モジュール図 (Module)module-info を選択—requires / exports / opens
コンポーネント図 (Component)図種切替-dAndroid コンポーネント構成
Manifest 図図種切替-MApplication + 各コンポーネント
Gradle 依存グラフ図種切替-Gモジュール / Maven 依存

CLI で図を出力する

引数に出力系オプションを付けると GUI を開かずに図やレポートを書き出します。出力拡張子で形式が決まります（.svg 推奨 / .png / .puml / .md）。以下の例は jar 版で書いていますが、java -jar Juml.jar を Juml.exe に置き換えれば exe 版でも同じです。

実コマンド例

# クラス図を SVG で出力
java -jar Juml.jar -c -o class.svg ~/AndroidStudioProjects/MyApp

# 特定メソッドのシーケンス図（Class.method を指定）
java -jar Juml.jar -q com.example.MainActivity.onCreate -o seq.svg ~/MyApp

# どの Class.method を -q に渡せるか一覧する
java -jar Juml.jar --list-methods ~/MyApp

# Gradle 依存グラフ
java -jar Juml.jar -G -o deps.svg ~/MyApp

# AndroidManifest 図
java -jar Juml.jar -M -o manifest.svg ~/MyApp

# 全成果物を一括出力（summary.md + 各種 SVG をディレクトリへ）
java -jar Juml.jar --all -o ./out ~/MyApp

# プロジェクト全体の Markdown サマリーをディレクトリへ
java -jar Juml.jar --summary -o ./out ~/MyApp

# フォルダ単位でクラス図を分割出力（サブフォルダ構造を維持）
java -jar Juml.jar -c -P -o ./out ~/MyApp

# 影響範囲（逆参照）の解析：あるシンボルを誰が使っているか
java -jar Juml.jar --impact com.example.Repo.save ~/MyApp

# クリック可能なクラスリンクを埋め込んだ SVG（ブラウザで探索向け）
java -jar Juml.jar -c --interactive-svg -o class.svg ~/MyApp

主要オプション早見表

よく使うものを抜粋します。全オプションは --help または 開発者リファレンスの CLI 節を参照してください。

オプション意味
-o FILE出力先ファイル / ディレクトリ（一括系はディレクトリ）
-c / -q M / -d / -M / -Gクラス / シーケンス / コンポーネント / Manifest / 依存グラフ
-A --all / --summary全成果物一括 / Markdown サマリー（いずれもディレクトリ出力）
-Q / -P --per-folderライフサイクル起点のシーケンス一括 / フォルダ単位クラス図
--seq-depth Nシーケンスのトレース深度（既定 5、0=無制限）
--exclude-external / --exclude-package PREFIX外部ライブラリ除外 / パッケージ除外（複数可）
--public-only / --no-fields / --no-methods表示メンバーの絞り込み
--preset minimal|balanced|detailedクラス図の見やすさプリセット（既定 balanced）
--relation inherit,impl,use関係線の種別を限定（CSV）
--list-methods / --function-listClass.method 候補一覧 / 全メソッド一覧（呼び出し元・条件付き）
--impact SYMBOL / --ref-find SYMBOL逆参照の影響解析 / 参照箇所の grep 向け列挙
-v --verbose / -h --help警告・サマリーを stderr に / ヘルプ表示
オプション記法の制約: 短縮形の連結（-vc）や --opt=value 形式は非対応です。値は --opt value と空白区切りで指定してください（例外は index サブコマンドの --db=path のみ）。複数モードを同時指定した場合は先に判定されたモードが優先されます。

index サブコマンド（任意・大規模向け）

大規模プロジェクトを繰り返し解析する場合は、SQLite インデックスを事前構築しておくと横断クエリ（参照関係など）が速くなります。詳細は 開発者リファレンスへ。

# プロジェクトのインデックスを構築（DB パスは --db= で指定可）
java -jar Juml.jar index ~/MyApp

エクスポート（SVG / PNG / PUML / MD）

形式用途メモ
.svg図の標準出力（推奨）ベクター・解像度無制限。巨大な図でも破綻しない
.png資料への貼り付け4096×4096 px 制限あり。大きい図は SVG を使う
.pumlPlantUML ソース手で微調整・版管理したいとき
.md / .markdownサマリーGradle / Manifest / プロジェクト全体の表・リスト

GUI では図を右クリック → Save As、または File メニューから保存します。CLI は -o の拡張子で形式が決まり、--all / --summary / -P はディレクトリへ複数ファイルを書き出します。

迷ったら SVG。 解像度無制限でクラス数が多くても潰れません。PNG は 4096px を超えると切れるため、大きな図は SVG で出してブラウザやベクターツールで閲覧してください。

便利機能

  • スコープ絞り込み — ツールバーの「Scope」/ Diagram → Scope。パッケージ / モジュール / 正規表現で対象を限定し、seed からの hop 距離や外部ライブラリ除外を指定して図を小さく保てます。
  • エンティティ検索 — Ctrl+Shift+F でクラス・メソッド・フィールドを横断検索し、結果から該当図へジャンプ。
  • スタイル / テーマ — Style メニューで PlantUML テーマ・フォント・コメント配置（inline / note）を切替。Custom Skinparam で skinparam を直接編集（!include 等のディレクティブは安全のため自動除去）。
  • ズーム・パン — Ctrl+= / Ctrl+- / Ctrl+0（100%）/ Ctrl+1（Fit）、マウスドラッグでパン。
  • Impact 解析 — あるクラス / メソッドの逆参照（影響範囲）を図と表で確認。CLI では --impact。
  • 設定の自動保存 — スタイルや最近開いたプロジェクトは自動で永続化（次回起動時に復元）。

よくあるワークフロー

1 クラスの周辺だけ見たい

GUI で対象クラスをクリック（seed + 1 hop のクラス図）。CLI なら関係を絞る:

java -jar Juml.jar -c \
  --exclude-external \
  -o focus.svg ~/MyApp

あるメソッドの呼び出しを追う

まず候補を一覧し、シーケンス図を出す:

java -jar Juml.jar --list-methods ~/MyApp
java -jar Juml.jar -q com.example.A.run \
  --seq-depth 4 -o run.svg ~/MyApp

プロジェクト全体を棚卸し

サマリー + 主要図をまとめて出力:

java -jar Juml.jar --all -o ./out ~/MyApp

変更の影響範囲を調べる

あるシンボルを誰が使っているか:

java -jar Juml.jar \
  --impact com.example.Repo.save ~/MyApp

トラブルシューティング

症状対処
jar が起動しない / バージョンエラーjava -version で 17 以上か確認。古い場合は JDK/JRE 17+ を入れるか、Java 不要の exe 版を使う。
./gradlew jpackageImage で「jpackage が見つかりません」JRE ではなく JDK 17+ が必要。JAVA_HOME を JDK に向ける。
Windows exe を Linux/macOS で作りたい不可。jpackage は実行 OS 向けのみ生成。Windows 上または CI の Windows ランナーで生成する。
図が大きすぎて描画されない / PNG が切れるスコープで対象を絞る、--exclude-external / --public-only で要素削減、出力は SVG を使う。詳細は GUI マニュアルの該当節。
大規模で OutOfMemoryjar 起動時にヒープを増やす: java -Xmx4g -jar Juml.jar …
解析に時間がかかる--mode headers-only の軽量パース、index サブコマンドで事前インデックス構築を検討。
